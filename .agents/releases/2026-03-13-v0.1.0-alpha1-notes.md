# Release v0.1.0-alpha1

**Date:** 2026-03-13

## Highlights

This is the first alpha release of the `edgenode` container — a production-ready Docker image for running Holochain applications as an always-on edge node.

## What's New

**s6-overlay process supervision** — The container now uses s6-overlay v3.2.0.2 as its init system. The Holochain conductor, log-sender, and logrotate are each managed as supervised services with proper startup ordering, automatic restart on failure, and clean shutdown semantics.

**Weave hApp management via wdocker** — The `@theweave/wdocker` CLI is bundled in the image, enabling management of Weave hApps directly from within a running container.

**Iroh networking** — The container ships with an iroh conductor config template for Holochain >= 0.6.1, enabling the newer peer-to-peer transport layer.

**Multi-architecture support** — Images are built for both `amd64` and `arm64`, with arch-specific binaries for s6-overlay and log-sender.

## All Changes

### Added
- s6-overlay v3.2.0.2 process supervisor replacing tini
- `@theweave/wdocker` CLI for Weave hApp management
- Iroh networking support for HC 0.6.1+
- Multi-arch (amd64/arm64) builds

### Changed
- Single consolidated `edgenode` image on HC 0.6.1-rc.3 + log-sender v0.1.5
- `happ_config_file` v0.3.0 with iroh as default and `priceSheetHash` support
- Kando webhapp fixture updated to v0.17.1
- Expanded quickstart documentation

### Fixed
- `log-sender.log` added to logrotate config
- log-sender stopped before `register-dna` to avoid file lock panic
- HC 0.6.1 conductor config `signal_url` and `relay_url` fields
- `happ_tool` `--help` and missing-args handling
- Integration data pipeline tests aligned with actual log-sender behaviour
